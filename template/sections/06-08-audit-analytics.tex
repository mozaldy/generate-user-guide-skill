\section{Audit \& Analytics}

\begin{ugModule}{Overview}

  \textbf{Purpose:}
  Track every document interaction with comprehensive
  device-level telemetry for governance, compliance, and
  incident investigation.

  \textbf{Who Can Access:}
  \ugRole{Admin} \ugRole{Manager} (full); other roles see
  limited personal data.
\end{ugModule}

% ────────────────────────────────────────────────────────────────
\subsection{Document Activity Log}

Every significant interaction with a document is recorded as
a \texttt{DocumentActivity} entry. The system tracks the
following activity action codes:

\begin{tabularx}{\textwidth}{@{} l l X @{}}
  \toprule
  \textbf{Action Code} & \textbf{Category}
    & \textbf{Description} \\
  \midrule
  \texttt{view}
    & Access
    & Document was previewed in the browser \\
  \texttt{download}
    & Access
    & Document was downloaded (watermarked PDF) \\
  \texttt{upload-document}
    & Content
    & A new document was created or a revision
      was uploaded \\
  \texttt{approve-document}
    & Workflow
    & Document was approved at a specific
      approval level \\
  \texttt{reject-document}
    & Workflow
    & Document was rejected at a specific
      approval level \\
  \texttt{request-access}
    & Access Control
    & A user submitted an access request for
      this document \\
  \texttt{approve-access}
    & Access Control
    & An access request was approved \\
  \texttt{reject-access}
    & Access Control
    & An access request was rejected \\
  \bottomrule
\end{tabularx}

% ────────────────────────────────────────────────────────────────
\subsection{Activity Telemetry Data}

Each activity record captures detailed telemetry about the
user's session:

\begin{tabularx}{\textwidth}{@{} l X @{}}
  \toprule
  \textbf{Field} & \textbf{Description} \\
  \midrule
  User Identity
    & Full name, user ID, and account reference \\
  Document \& Version
    & Which document and which version were
      accessed \\
  Action Code
    & The type of action performed (see table
      above) \\
  Browser
    & The browser name (e.g., Chrome, Firefox,
      Edge) \\
  Operating System
    & The user's OS (e.g., Windows 11, macOS,
      Android) \\
  Device
    & Device type (e.g., Desktop, Mobile,
      Tablet) \\
  IP Address
    & Client IP address at time of action \\
  User Agent
    & Full user agent string for detailed
      client identification \\
  Watermark ID
    & Unique watermark identifier embedded in
      the downloaded PDF (links the physical
      document back to this activity record) \\
  Request ID
    & Unique request identifier for
      cross-referencing with server logs \\
  Date
    & Calendar date of the activity \\
  Year
    & Calendar year (for partitioning and
      reporting) \\
  Timestamp
    & Exact date and time of the action \\
  Status
    & Success or failure indicator \\
  \bottomrule
\end{tabularx}

\begin{ugImportant}
  The \texttt{WatermarkId} field creates a forensic link
  between a physical downloaded PDF and the database activity
  record. If a watermarked document is found outside the
  organization, the watermark ID can be traced back to the
  exact user, time, device, and IP address of the download.
\end{ugImportant}

\subsection{Document Statistics}

The \ugMenu{Stats} section on a document detail page shows:

\begin{itemize}[leftmargin=18pt]
  \item \textbf{Total Views} --- number of preview events
  \item \textbf{Total Downloads} --- number of download events
  \item \textbf{Unique Users} --- distinct users who accessed
        the document
  \item \textbf{Last Access} --- timestamp of the most recent
        activity
\end{itemize}

The stats API (\texttt{GET /api/v1/document/\{id\}/stats})
returns aggregated data for \ugRole{Admin} and \ugRole{Manager}
roles.

% ────────────────────────────────────────────────────────────────
\subsection{Document Timeline}

The \ugMenu{Timeline} section on a document detail page
presents a unified, chronological feed of all activities
related to that document. The timeline API
(\texttt{GET /api/v1/document/\{id\}/timeline}) returns:

\begin{itemize}[leftmargin=18pt]
  \item Activity type with icon and color coding
  \item User who performed the action
  \item Device and browser information
  \item Timestamp with relative time display
  \item Version number that was accessed
\end{itemize}

% ────────────────────────────────────────────────────────────────
\subsection{Dashboard Analytics Charts}

\begin{itemize}[leftmargin=18pt]
  \item \textbf{Download Trend Chart} --- Line/area chart
        showing daily download volume over the last 30~days
  \item \textbf{View Trend Chart} --- Line/area chart
        showing daily preview volume over the last 30~days
\end{itemize}

These charts are displayed on the document detail page and
provide at-a-glance insight into document popularity and
usage patterns.

% ────────────────────────────────────────────────────────────────
\subsection{Audit Data Retention}

\begin{itemize}[leftmargin=18pt]
  \item Activity logs are retained \textbf{indefinitely} and
        are never hard-deleted.
  \item Soft-deleted activity records retain their
        \texttt{DeletedDate} and \texttt{DeletedBy} fields
        for traceability.
  \item Each record includes standard audit fields:
        \texttt{CreatedDate}, \texttt{CreatedBy},
        \texttt{UpdatedDate}, \texttt{UpdatedBy}.
  \item Activity data is partitioned by date and year for
        efficient querying of historical records.
\end{itemize}

\begin{ugWarning}
  Audit logs are critical for compliance and incident
  investigation. If you suspect unauthorized access, contact
  your administrator immediately. Activity logs provide full
  traceability including device fingerprint, IP address, and
  watermark ID linking to the physical document.
\end{ugWarning}
